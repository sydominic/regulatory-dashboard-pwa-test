# V1.8 Local Collector All Sources + Query Only UI

## 목적
- v1.7 기준 제목 상세페이지 확정 로직은 유지한다.
- 로컬 수집기가 안내서/지침(m_218), 학술토론회(m_220), 전문홍보물(m_231)까지 반드시 시도하도록 수정했다.
- Render 화면에서는 빠른수집/기간수집 버튼을 제거하고, 기간 설정 및 조회 기능만 남겼다.

## 핵심 변경
1. 수집 대상 누락 방지
   - MFDS_SOURCES에는 이미 m_218, m_220, m_231이 있었으나, 전역 상세검증 제한이 m_1060 단계에서 걸리면 이후 게시판까지 도달하지 못했다.
   - v1.8에서는 상세검증 제한을 전체 전역이 아니라 게시판별 제한으로 적용한다.
   - 한 게시판이 detail limit에 도달해도 전체 수집 루프를 중단하지 않고 다음 게시판으로 진행한다.

2. 화면 버튼 정리
   - Render에서 식약처 직접수집은 timeout으로 부적합하므로 빠른수집/기간수집 버튼을 제거했다.
   - 기간 선택, 직접기간 설정, 검색어, 조회 버튼은 유지했다.

3. 유지되는 구조
   - 로컬 수집기 → Supabase 저장
   - Render 대시보드 → Supabase 조회
   - 다른 사용자는 Render URL 조회만 수행

## 확인 포인트
- 로컬 수집 로그에 아래 3개가 보여야 한다.
  - 게시판 시작: 안내서/지침 (m_218)
  - 게시판 시작: 학술토론회 (m_220)
  - 게시판 시작: 전문홍보물 (m_231)
- 해당 게시판에 기간 내 게시물이 없으면 최종 0으로 표시되는 것이 정상이다.
